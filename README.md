# ssh_relay

`ssh_relay.py` — небольшой локальный SSH-relay для выполнения удалённых команд через одну уже открытую SSH-сессию.

Основной сценарий использования — работа CLI-агента, например OpenCode, с удалённым сервером, где вход выполняется по SSH-паролю. Пользователь один раз запускает relay и вводит пароль вручную, после чего агент может выполнять короткие удалённые команды через локальный вызов `ssh_relay.py exec`, не запрашивая пароль заново.

## Возможности

* установление SSH-соединения с удалённым сервером по паролю;
* сохранение открытой SSH-сессии на время работы daemon-процесса;
* выполнение удалённых shell-команд через локальный relay;
* просмотр состояния активной сессии;
* завершение сессии отдельной командой;
* привязка локального служебного порта только к `127.0.0.1`.

## Ограничения

Текущая версия предназначена для коротких неинтерактивных команд.

Не поддерживаются:

* передача файлов по SCP/SFTP;
* интерактивные программы на удалённом сервере;
* передача пароля в `sudo`, `passwd` и аналогичные команды;
* параллельное выполнение нескольких удалённых команд;
* безопасная проверка SSH host key: неизвестный ключ сервера принимается автоматически.

Для команд, которым необходим `sudo`, на сервере должен быть заранее настроен подходящий неинтерактивный способ выполнения.

Не рекомендуется использовать relay для команд с очень большим объёмом вывода, например чтения крупных логов целиком или полного рекурсивного обхода файловой системы.

## Требования

* Python 3;
* библиотека `paramiko`;
* сетевой доступ к SSH-серверу;
* учётная запись SSH с парольной аутентификацией.

## Установка зависимости

```powershell
py -m pip install paramiko
```

## Команды

```text
python ssh_relay.py daemon --host HOST --user USER [--port PORT]
python ssh_relay.py exec "COMMAND"
python ssh_relay.py status
python ssh_relay.py stop
```

### `daemon`

Открывает SSH-соединение с сервером и запускает локальный relay-процесс.

```powershell
py .\ssh_relay.py daemon --host 192.0.2.10 --user deploy
```

При запуске будет запрошен SSH-пароль:

```text
SSH password for deploy@192.0.2.10:
```

После успешного подключения relay выводит параметры активной сессии:

```text
Connected: deploy@192.0.2.10:22
Daemon listening on 127.0.0.1:54321
Press Ctrl+C to stop.
```

Окно терминала с запущенным daemon должно оставаться открытым до завершения работы.

Нестандартный SSH-порт задаётся параметром `--port`:

```powershell
py .\ssh_relay.py daemon --host 192.0.2.10 --port 2222 --user deploy
```

### `exec`

Выполняет одну команду на удалённом сервере через активную SSH-сессию.

```powershell
py .\ssh_relay.py exec "hostname"
py .\ssh_relay.py exec "whoami"
py .\ssh_relay.py exec "uname -a"
```

Пример выполнения команды в каталоге проекта:

```powershell
py .\ssh_relay.py exec "cd /opt/project && git status --short"
```

Код завершения удалённой команды возвращается вызывающему процессу. Стандартный вывод удалённой команды направляется в stdout, диагностический вывод — в stderr.

### `status`

Проверяет наличие активной relay-сессии.

```powershell
py .\ssh_relay.py status
```

Пример вывода для работающей сессии:

```text
Active: deploy@192.0.2.10:22
Port: 54321
```

### `stop`

Завершает daemon-процесс и удаляет данные текущей сессии.

```powershell
py .\ssh_relay.py stop
```

Daemon также можно завершить нажатием `Ctrl+C` в терминале, где он был запущен.

## Файл сессии и рабочий каталог

Relay сохраняет параметры текущей сессии в файл:

```text
.ssh_relay_session.json
```

Файл создаётся в текущем рабочем каталоге, из которого был запущен `daemon`.

Поэтому команды `daemon`, `exec`, `status` и `stop` необходимо выполнять из одного и того же каталога.

Пример:

```powershell
cd C:\Tools\ssh-relay
py .\ssh_relay.py daemon --host 192.0.2.10 --user deploy
```

Во втором терминале:

```powershell
cd C:\Tools\ssh-relay
py .\ssh_relay.py status
py .\ssh_relay.py exec "hostname"
```

При запуске `exec` из другого каталога скрипт не найдёт активную сессию.

Файл `.ssh_relay_session.json` содержит локальный токен доступа к открытой SSH-сессии. Его не следует публиковать, добавлять в Git или передавать другим пользователям.

Рекомендуется добавить его в `.gitignore`:

```gitignore
.ssh_relay_session.json
```

## Использование с OpenCode

Перед началом работы необходимо вручную запустить daemon:

```powershell
cd C:\Tools\ssh-relay
py .\ssh_relay.py daemon --host 192.0.2.10 --user deploy
```

После этого агенту следует указать, что прямое подключение по SSH использовать не нужно, а все удалённые команды должны выполняться через relay.

Пример инструкции для агента:

```text
Удалённый сервер доступен через уже запущенный локальный SSH relay.

Не используй прямые вызовы ssh и не запрашивай пароль.
Для каждой команды на удалённом сервере сначала переходи в каталог relay:

cd C:\Tools\ssh-relay

Затем выполняй удалённую команду так:

py .\ssh_relay.py exec "<remote-command>"

Перед началом работы проверь подключение командами:

py .\ssh_relay.py status
py .\ssh_relay.py exec "hostname && whoami && pwd"

Не запускай интерактивные команды, команды с запросом sudo-пароля и команды, создающие очень большой вывод.
```

## Пример рабочей сессии

Первый терминал:

```powershell
cd C:\Tools\ssh-relay
py .\ssh_relay.py daemon --host 192.0.2.10 --user deploy
```

Второй терминал:

```powershell
cd C:\Tools\ssh-relay

py .\ssh_relay.py status
py .\ssh_relay.py exec "hostname"
py .\ssh_relay.py exec "cd /opt/project && git status --short"
py .\ssh_relay.py exec "cd /opt/project && python3 -m py_compile server.py"
py .\ssh_relay.py stop
```

## Замечания по безопасности

Текущая версия имеет следующие особенности безопасности:

* SSH-пароль вводится вручную и не записывается в файл сессии;
* relay слушает только локальный адрес `127.0.0.1`;
* токен выполнения команд хранится в `.ssh_relay_session.json`;
* неизвестный SSH host key автоматически принимается при подключении.

Последний пункт означает, что текущую реализацию следует использовать только в контролируемой среде. Для более безопасного применения необходимо добавить проверку host key сервера через `known_hosts` или заранее заданный fingerprint.
